# Human IQ Implementation Start Standard

## Purpose
Define what counts as project start, how the first delivery window should be handled, and when ongoing optimization should be recommended after initial work.

## What project start means
Treat a project as started when all of these are true:

1. The engagement has been accepted.
2. The required deposit or first payment has been received.
3. The minimum onboarding inputs needed to begin are in hand.
4. The first milestone or first phase is confirmed internally.

Do not treat verbal enthusiasm or partial materials as project start.

## Start standard by engagement type
### Growth & Conversion Audit
Project start means:
- payment is in
- required review materials are received
- Human IQ can begin diagnosis work

### Strategy + Implementation Sprint
Project start means:
- deposit is in
- first-phase scope is confirmed
- any required access or materials for the sprint are available

### Ongoing Optimization Partner
Project start means:
- first monthly payment is in
- first 30-day focus is confirmed
- the starting review cadence is set

## Stabilization window
Use a short stabilization window after initial delivery or implementation.

The stabilization window is for:
- confirming the client understands what was delivered
- catching immediate issues tied to the agreed scope
- making light cleanup adjustments where appropriate

The stabilization window is not for:
- adding new scope informally
- extending delivery indefinitely
- replacing a new paid phase with unpaid extras

## Handoff into ongoing care or optimization
After the initial engagement:

1. confirm what changed
2. confirm what remains
3. identify whether there is enough remaining value for ongoing support
4. recommend the next phase only if there is a real business case

## When to recommend Ongoing Optimization Partner after a sprint
Recommend it when most of these are true:

- the first sprint solved only the first meaningful layer
- the business has ongoing demand and recurring optimization opportunities
- lead flow, qualification, follow-up, or conversion still need continued refinement
- the client would benefit from regular review and adjustment
- Human IQ can clearly create further measurable value

Do not recommend it automatically just because the sprint is ending.

## Operator rule
Before any implementation start:
- confirm scope
- confirm payment
- confirm inputs
- confirm first action

Lean execution is better than a premature kickoff.
