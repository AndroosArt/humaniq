# Human IQ Onboarding And Maintenance

## Purpose
Define what happens after a prospect says yes so acceptance leads cleanly into onboarding, delivery, and any ongoing optimization path.

## Post-acceptance flow
1. Acceptance is received
2. Deposit or first payment is collected if applicable
3. Onboarding message is sent
4. Required client inputs are collected
5. Kickoff happens only if useful
6. Implementation or audit work starts
7. Stabilization window is completed
8. Ongoing optimization path is offered when appropriate

## Acceptance
Acceptance can be confirmed through:

- signed proposal
- written email approval
- paid invoice or deposit

Do not treat a lead as accepted until the commercial yes is clear.

## Deposit or first payment
Use the simplest viable payment step for now.

- Audit: collect first payment before the work begins
- Sprint: collect deposit before start
- Ongoing partner: collect first monthly payment before activation

## Onboarding
Send one concise onboarding email that includes:

- confirmation of the engagement path
- what happens next
- what is needed from the client
- timeline for the first milestone
- payment reminder if still outstanding

## Required onboarding inputs
Collect only what is needed to start:

- primary point of contact
- access or documents required for review
- current tools or systems in scope
- any key business context needed for the first phase

## Kickoff
Kickoff is optional.

Use a kickoff call only when:
- stakeholder alignment is needed
- scope needs live confirmation
- access or workflow complexity is high

If kickoff is not needed, start asynchronously with a written onboarding note and first action list.

## Implementation start
### Audit
- confirm scope
- review materials
- produce findings and priorities
- deliver the Practical AI Blueprint if that is the agreed output

### Sprint
- confirm first-phase scope
- start implementation
- keep communication concise and outcome-focused

### Ongoing partner
- confirm the first 30-day focus
- set the review cadence
- begin optimization work

## Stabilization window
After delivery or first implementation:

- confirm the client understands what changed
- capture any immediate issues
- make limited cleanup adjustments tied to the agreed scope
- avoid endless informal add-ons

## Ongoing care and optimization model
Use a simple ongoing model:

### Review cadence
- monthly by default
- more frequent only if justified by engagement scope

### Focus areas
- lead flow quality
- qualification quality
- follow-up performance
- conversion friction
- practical systems improvement

### Ongoing outputs
- short review summary
- recommended next improvements
- agreed work for the next cycle

## Upgrade logic
### Audit to Sprint
Offer when the diagnosis is clear and the highest-value first phase is defined.

### Sprint to Ongoing Partner
Offer when the business would benefit from continuing refinement after the first implementation phase.

### Ongoing Partner continuation
Continue only if Human IQ is still creating clear business value.
